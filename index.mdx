Welcome to the official technical and operational documentation for the **Sasini Mobile Application**. 

This documentation serves as a comprehensive blueprint. It details the precise user interfaces, admin console capabilities, and backend API interactions that power our digital agricultural ecosystem.

<Note>
  **Current Scope (Phase 1):** The application and this documentation are currently fully deployed for **Tea Farmers**. Subsequent phases will integrate Coffee, Macadamia, and Avocado pipelines.
</Note>

## Platform Capabilities by Role

The Sasini ecosystem is role-based, ensuring that every user interacts only with the data and tools relevant to their position. Choose your perspective below to understand how the platform serves different stakeholders.

<Tabs>
  <Tab title="👨‍🌾 For Farmers">
    **The Mobile Experience**
    Farmers use the mobile application to digitize their daily operations.
    * **Green Leaf Tracking:** Real-time visibility into daily weighbridge deliveries.
    * **Financial Transparency:** Instant access to payment calculations and digital receipts.
    * **Digital Extension:** AI-powered agronomic advice and real-time alerts.
  </Tab>
  <Tab title="🏢 For Administrators">
    **The Web Console**
    Factory managers and Extension Officers utilize the admin dashboard for macro-level oversight.
    * **Yield Aggregation:** Live tracking of daily factory intake across all collection centers.
    * **Farmer Verification:** Validating and managing the digital profiles of registered outgrowers.
    * **Discrepancy Resolution:** Flagging and resolving weight anomalies.
  </Tab>
  <Tab title="💻 For Developers">
    **The Technical Stack**
    Engineers maintaining or integrating with the Sasini platform.
    * **Easyway Web API:** Secure data exchange protocols for farmer validation and payment retrieval.
    * **MFA & Encryption:** Implementation details of our Multi-Factor Authentication and data hashing.
    * **Payload Structures:** Expected JSON schemas for all major endpoints.
  </Tab>
</Tabs>
